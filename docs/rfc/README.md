# Request for Comments (RFC)

This directory contains formal proposals for changes to the Open Financial Network (OFN).

RFCs are used to:
- Propose new features or protocols
- Modify existing system behavior
- Define standards and interfaces
- Document governance and consensus decisions

## RFC Lifecycle

1. Draft – Proposal is under discussion
2. Review – Community and maintainers review
3. Accepted – Proposal is approved for implementation
4. Implemented – Changes are merged into core
5. Final – RFC is locked and archived

## RFC Naming

RFCs follow the format:

RFC-0001.md  
RFC-0002.md  

Lower numbers represent foundational decisions.

## Authority

Acceptance of an RFC requires:
- Technical feasibility
- Security review
- Alignment with OFN principles
